关键目录/文件（简短）
- src/: 浏览器 SPA
  - src/app/: 应用编排（data-loader、刷新、布局）
  - src/components/: 面板组件（扩展 Panel）
  - src/services/: 业务逻辑
  - src/config/: 面板/variant 配置（src/config/variants/）
  - src/generated/: proto 生成的客户端/服务端桩
  - src/workers/: Web Worker（分析/ML/向量 DB）
- api/: Vercel Edge Functions（自包含 JS, 不能 import server/src）
- server/_shared/: Redis、限流、LLM helper、cachedFetchJson()
- server/worldmonitor/<domain>/: RPC 处理器（与 proto 对应）
- proto/: protobuf 定义（改 proto → make generate → 更新 src/generated）
- src-tauri/sidecar/: 桌面 Node sidecar（桌面智能体放这里）
- api/bootstrap.js & scripts/: 数据引导/种子

添加新智能体（推荐流程）
1. 确定位置：后端服务（server + api）、前端面板（src/components）或桌面 sidecar（src-tauri/sidecar）。
2. 若需新 API：在 proto/worldmonitor/<domain>/ 加定义并添加 (sebuf.http.config) RPC，运行 make generate，实现在 server/worldmonitor/<domain>/ 的 handler，并在需要时在 api/<domain>/ 建 Edge 函数作暴露。
3. 若需 UI：新建 src/components/MyAgentPanel.ts（继承 Panel），在 src/config/panels.ts 注册，并在 src/app/data-loader.ts 接入数据源。
4. 与 LLM 交互：复用 server/_shared/LLM helper 与 cachedFetchJson（含 Redis 缓存/防雪崩）。
5. 更新 variant（src/config/variants/）、测试（tests/ 或 e2e/），在本地跑 npm install，然后 npm run dev 测试；若改 proto，执行 make generate 并运行相关测试。
